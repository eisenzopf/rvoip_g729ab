- DecoderState struct: complete field list with exact initial values (PRD §2.2.1), including `sid_sav=0`, `sh_sid_sav=1`, `seed_fer=21845`, `seed=11111`, `prev_ma=0` (previous MA predictor mode for frame erasure LSP recovery; `LSPDEC.C`), `sid_gain=tab_Sidgain[0]=2` (initial CNG gain; `DEC_SID.C`), post-filter buffers: `res2_buf[PIT_MAX+L_SUBFR]=0`, `scal_res2_buf[PIT_MAX+L_SUBFR]=0`, `mem_syn_pst[M]=0`, `mem_pre=0` (tilt compensation preemphasis memory; `POSTFILT.C:351`. **Persistence note:** `mem_pre` is a function-local `static` inside `preemphasis()`, NOT file-scope. `Init_Post_Filter()` does NOT reset it — it persists from program start in C. In Rust, `DecoderState::new()` should initialize to 0, and `reset()` should also zero it for clean state, even though the C code's `Init_Post_Filter()` does not touch it), `past_gain=4096` (Q12 AGC smoothed gain; `POSTFILT.C`. **Verified:** `past_gain` is a function-local `static` inside `agc()` in `postfilt.c:384`, explicitly declared as `static Word16 past_gain=4096;` with comment `/* past_gain = 1.0 (Q12) */`. - **`Init_Post_Filter()` reset scope** (`postfilt.c:60-77`): The C function explicitly zeros `res2_buf[PIT_MAX+L_SUBFR]`, `scal_res2_buf[PIT_MAX+L_SUBFR]`, `mem_syn_pst[M]`, and the local synthesis buffer `synth_buf[L_FRAME+M]`. It does NOT reset `mem_pre` (tilt preemphasis memory, function-local static in `preemphasis()` at line 351) or `past_gain` (AGC smoothed gain, function-local static in `agc()` at line 384 = 4096 Q12). In Rust, `DecoderState::new()` must initialize all fields (including `mem_pre=0` and `past_gain=4096`). A `reset()` method, if provided, should match `Init_Post_Filter()`'s scope — zeroing only `res2_buf`, `scal_res2_buf`, `mem_syn_pst`, and the synthesis buffer, while leaving `mem_pre` and `past_gain` untouched. This distinction matters for mid-session reset correctness where the C program would call `Init_Post_Filter()` without reinitializing the function-local statics
- **`synth_buf[L_FRAME+M]` in `Post_Filter()` is a local scratch array, NOT persistent state** (`postfilt.c:101-103`): In the C code, `synth_buf` is declared as a function-local `static` inside `Post_Filter()`. At the start of each call, `Post_Filter()` copies `mem_syn_pst[M]` into `synth_buf[0..M]` as the synthesis filter warm-up, then writes the full-frame synthesis output into `synth_buf[M..M+L_FRAME]`, and finally copies the last `M` samples back to `mem_syn_pst`. The persistent data is fully captured by `mem_syn_pst` — `synth_buf` merely provides scratch space for the in-place synthesis operation. In Rust, `synth_buf` should be a **local array** within the post-filter pipeline function (e.g., `let mut synth_buf = [0i16; L_FRAME + M];`), not a field of `DecoderState`. This avoids unnecessary state bloat and makes the data flow explicit

- LP synthesis overflow handling: trial synthesis, overflow detected, excitation >>2, re-synthesize
- Frame erasure concealment: gain decay (0.9x pitch AND cap at 0.9, 0.98x code), pitch continuation **per-subframe** (old_T0 increments by 1 per subframe, net +2 per erased frame)
- **`sharp` update during active subframes:** After gain decoding in each active subframe, the decoder updates `sharp = shr(gain_pitch, 1)` if `gain_pitch < SHARPMAX`, else `sharp = SHARPMAX`; floor at `SHARPMIN`. This ensures pitch sharpening uses the current frame's adaptive codebook gain, not a stale value. Without this per-subframe update, the fixed codebook vector sharpening applied before `Pred_lt_3()` would use incorrect gains. Reference: `DEC_LD8A.C:316-321`
- **`Gain_predict` complete algorithm** (`gainpred.c:29-91`): MA prediction of innovation energy, producing `gcode0`/`exp_gcode0` for the gain codebook search. Fixed-point pipeline: (1) Energy: `L_tmp = 0; for i in 0..L_subfr: L_tmp = L_mac(L_tmp, code[i], code[i])` (Q13*Q13→Q27). (2) Log domain: `Log2(L_tmp, &exp, &frac)` extracts exponent (Q0) and fraction (Q15). (3) Mean-removed energy: `L_tmp = Mpy_32_16(exp, frac, -24660)` where -24660 is -3.0103 in Q13, giving result in Q14; then `L_tmp = L_mac(L_tmp, 32588, 32)` adds 127.298 (=30.0-10*log10(L_subfr)+10*log10(2^27)) in Q14. (4) MA prediction: `L_tmp = L_shl(L_tmp, 10)` (Q14→Q24); `for i in 0..4: L_tmp = L_mac(L_tmp, pred[i], past_qua_en[i])` (Q13*Q10→Q24). (5) Extract: `gcode0 = extract_h(L_tmp)` (Q24→Q8). (6) Linear domain: `L_tmp = L_mult(gcode0, 5439)` (0.166 in Q15, result in Q24); `L_tmp = L_shr(L_tmp, 8)` (Q24→Q16); `L_Extract(L_tmp, &exp, &frac)`; `gcode0 = extract_l(Pow2(14, frac))`; `exp_gcode0 = 14 - exp`. Output: `gcode0` in Q(`exp_gcode0`), range 16768..32767. Used by both encoder (gain codebook search) and decoder (gain decoding). Reference: `GAINPRED.C:29-91`
- **`Gain_update` signature and algorithm** (`gainpred.c:99-123`): `fn gain_update(past_qua_en: &mut [Word16; 4], l_gbk12: Word32)` — `l_gbk12` is `Word32` (Q13), NOT `Word16`. Pipeline: shift history (`past_qua_en[3..1] = past_qua_en[2..0]`); compute `past_qua_en[0] = 20*log10(l_gbk12)` in Q10 via: `Log2(l_gbk12, &exp, &frac)` → `L_acc = L_Comp(exp-13, frac)` (Q16) → `tmp = extract_h(L_shl(L_acc, 13))` (Q13) → `past_qua_en[0] = mult(tmp, 24660)` (24660=6.0205 in Q12, result in Q10). Reference: `GAINPRED.C:99-123`
- **`Gain_update_erasure` signature and algorithm** (`gainpred.c:137-158`): `fn gain_update_erasure(past_qua_en: &mut [Word16; 4])`. Pipeline: `L_tmp = sum(past_qua_en[0..4])` via `L_deposit_l` + `L_add`; `av = L_shr(L_tmp, 2)` (mean); `av = av - 4096` (-4.0 dB in Q10); clamp to floor -14336 (-14.0 dB in Q10); shift history and store `past_qua_en[0] = av`. Reference: `GAINPRED.C:137-158`
- Post-filter chain operates in the **spectrally shaped residual domain** (not directly on synthesized speech and not the plain LP residual). Per-subframe pipeline: (a) compute residual through `A(z/GAMMA2_PST)` (γ_n=0.55, numerator of H_f) via `Weight_Az(Az, GAMMA2_PST, Ap3); Residu(Ap3, synth, res2)` — NOT `A(z)`, (b) scale >>2 for overflow avoidance (`scal_res2[j]=shr(res2[j],2)`), (c) pitch PF on scaled residual (correlation search) / unscaled residual (output); bypass when `Vad==0`, (d) tilt compensation from combined post-filter impulse response `h_f` of `A(z/GAMMA2_PST)/A(z/GAMMA1_PST)` — applied to the residual-domain signal `res2_pst` BEFORE synthesis (Annex A spec A.4.2.3: "The compensation filtering H_t(z) is performed before synthesis through 1/Â(z/γ_d)"), (e) re-synthesize through `1/A(z/GAMMA1_PST)` (γ_d=0.70, denominator of H_f) via `Weight_Az(Az, GAMMA1_PST, Ap4); Syn_filt(Ap4, res2_pst, ...)` — NOT `A(z)`, (f) AGC. Constant mapping: `GAMMA2_PST=18022` (0.55 Q15) = γ_n, `GAMMA1_PST=22938` (0.70 Q15) = γ_d. Reference: `postfilt.c:120-200`
- **Post-filter tilt coefficient extraction algorithm** (`postfilt.c:154-180`): The tilt compensation coefficient `mu` is derived from the combined post-filter impulse response, not from the LP coefficients directly. Per-subframe pipeline: (1) Form impulse response of `A(z/gamma_n)`: `h[0..M] = Ap3[0..M]` (numerator coefficients), `h[M+1..L_H-1] = 0`. (2) Filter through `1/A(z/gamma_d)`: `Syn_filt(Ap4, h, h, L_H, &h[M+1], 0)` — this produces `h[0..L_H-1]` as the impulse response of `A(z/gamma_n)/A(z/gamma_d)`. (3) Autocorrelation: `r_h0 = extract_h(L_mult(h[0],h[0]) + SUM(L_mac(h[i],h[i]) for i=1..L_H-1))`, `r_h1 = extract_h(L_mult(h[0],h[1]) + SUM(L_mac(h[i],h[i+1]) for i=1..L_H-2))`. (4) Tilt coefficient: if `r_h1 <= 0` then `mu = 0` (no tilt compensation); else `mu = div_s(mult(r_h1, MU), r_h0)` where `MU = 26214` (0.8 in Q15). Note `gamma_t = 0.8` only when first-order reflection coefficient `k'_1 = r_h1/r_h0 > 0`; when `k'_1 <= 0`, `gamma_t` is effectively 0.0 (E11). (5) Apply: `preemphasis(res2_pst, mu, L_SUBFR)` implements `H_t(z) = 1 - mu*z^-1`. Reference: `POSTFILT.C:154-180`
- Post-filter bypass: `DecoderConfig::post_filter` flag propagated to `Post_Filter()`. ITU conformance tests require `post_filter=true`. Add a unit test verifying bypass mode produces unfiltered output
- DspContext in post-filter: the formant post-filter calls `Syn_filt` internally. While the overflow retry path (excitation >>2) only applies to the main decoder synthesis, `DspContext` must still be threaded through the post-filter chain. Clear the overflow flag before main synthesis to prevent false triggers from post-filter operations
- Post-filter residual buffer sliding-window pattern: `res2_buf[PIT_MAX+L_SUBFR]` and `scal_res2_buf[PIT_MAX+L_SUBFR]` use C pointer arithmetic (`res2 = res2_buf + PIT_MAX`) to create a sliding window where `res2[0..L_SUBFR]` is the current subframe and `res2[-PIT_MAX..-1]` is past data for pitch post-filter correlation. After each subframe, `Copy()` shifts the tail of the buffer backward (`postfilt.c:191-192`). Rust translation: use index-based access with a constant offset (`PIT_MAX`) instead of pointer arithmetic — e.g., `&res2_buf[PIT_MAX..PIT_MAX+L_SUBFR]` for the current subframe and `&res2_buf[PIT_MAX-t0_max..PIT_MAX]` for the pitch search window
- Excitation buffer: `old_exc` array of size `PIT_MAX + L_INTERPOL + L_FRAME = 143 + 11 + 80 = 234` samples. The first `PIT_MAX + L_INTERPOL = 154` samples hold past excitation; the remaining `L_FRAME = 80` are the current frame. This sizing is critical for the overflow retry path and for correct `Pred_lt_3` operation
- AGC zero-energy edge case (PRD §5.9.4): when post-filtered signal energy E_out is zero, set gain G to zero and reset adaptive gain state to 0; passes signal unchanged to prevent division by zero
- Post-processing HP filter state MUST use DPF split high/low precision (y1_hi, y1_lo, y2_hi, y2_lo)
- `sid_sav`/`sh_sid_sav` excitation energy computation runs on every good frame (needed by CNG in Phase 9)
- **`past_ftyp` update at end of decoder frame** (`dec_ld8a.c:371`): After the subframe loop and `sid_sav`/`sh_sid_sav` computation, the decoder executes `past_ftyp = ftyp`. This runs on every frame (both active and inactive) and must be the last state update before the function returns. Without this update, `past_ftyp` remains stale, causing incorrect BFI routing when Annex B is enabled (Phase 9): `bfi==1 && past_ftyp==1` triggers speech erasure concealment, while `bfi==1 && past_ftyp!=1` triggers DTX erasure (continue CNG)
- **Decoder `parm[]` array layout:** `Decod_ld8a(parm[], synth[], A_t[], T2[], &Vad)` receives parameters from `read_frame()` in `bits.c`. Layout: `parm[0]` = BFI (Bad Frame Indicator: 0=good frame, 1=erased) set by `read_frame()`; `parm[1]` = frame type (`ftyp`: 0=untransmitted, 1=speech, 2=SID) set by `bits2prm_ld8k()` inside `read_frame()`; `parm[2..12]` = 11 coded parameters for speech frames (per `bitsno[11]` = {8,10,8,1,13,4,7,5,13,4,7}, sums to 80 bits). The 11 packed parameter entries map to subfields as follows: `bitsno[0]`=8 is L0(1)+L1(7) LSP indices at `parm[2]`, `bitsno[1]`=10 is L2(5)+L3(5) LSP indices at `parm[3]`, `bitsno[2]`=8 is P1 pitch delay SF1 at `parm[4]`, `bitsno[3]`=1 is P0 parity at `parm[5]`, `bitsno[4]`=13 is C1 fixed CB SF1 at `parm[6]`, `bitsno[5]`=4 is S1 signs SF1 at `parm[7]`, `bitsno[6]`=7 is GA1(3)+GB1(4) gain SF1 at `parm[8]`, `bitsno[7]`=5 is P2 pitch delay SF2 at `parm[9]`, `bitsno[8]`=13 is C2 fixed CB SF2 at `parm[10]`, `bitsno[9]`=4 is S2 signs SF2 at `parm[11]`, `bitsno[10]`=7 is GA2(3)+GB2(4) gain SF2 at `parm[12]`; for SID frames `parm[2..5]` = 4 SID parameters (per `bitsno2[4]` = {1,5,4,5}, where `bitsno2[0]`=1 is L0 MA predictor mode at `parm[2]`, `bitsno2[1]`=5 is 1st-stage SID LSF VQ index at `parm[3]`, `bitsno2[2]`=4 is 2nd-stage SID LSF VQ index at `parm[4]`, `bitsno2[3]`=5 is SID gain index at `parm[5]`; total 15 bits). **BFI extraction inside `Decod_ld8a`:** The C code extracts BFI via `bfi = *parm++` (`dec_ld8a.c:147`), which reads `parm[0]` and advances the pointer so that subsequent accesses are offset by 1 from the original array start. In Rust, this maps to reading the first element of the parameter slice, then operating on the remaining slice. The decoder routes to speech/SID/erasure paths based on `(bfi, ftyp, past_ftyp)`. **Parity check note:** `read_frame()` calls `Check_Parity_Pitch(parm[4], parm[5])` for speech frames (`bits.c:232-235`), writing the parity check result back into `parm[5]` before `Decod_ld8a()` is called. The decoder receives a pre-checked parity value, not the raw parity bit. Reference: `BITS.C:108-238`, `DEC_LD8A.C:118-160`
- **`Decod_ld8a` output parameters for post-filter interface:** The C `Decod_ld8a` outputs not just `synth[L_FRAME]` but also `A_t[2*(M+1)]` (two sets of LP coefficients, one per subframe), `T2[2]` (two decoded pitch delays, one per subframe), and `Vad` (VAD flag for CNG). These are consumed by `Post_Filter(synth, A_t, T2, Vad)`. In Rust, these become either fields on `DecoderState` or a return struct passed from the decode step to `Post_Filter()`. `A_t` and `T2` must be per-frame scratch (not persistent state). Reference: `DECODER.C:130-133`, `DEC_LD8A.C:107-109`
- **Decoder excitation buffer shift at frame end:** After the subframe loop completes, the decoder shifts `old_exc[]` backward: `Copy(&old_exc[L_FRAME], &old_exc[0], PIT_MAX+L_INTERPOL)`, preserving 154 samples of past excitation history for the next frame's `Pred_lt_3()` sinc interpolation. In Rust: `old_exc.copy_within(L_FRAME.., 0)`. This is parallel to the encoder's buffer shifts documented in Phase 6. Reference: `DEC_LD8A.C:365-366`
- **`Decod_ACELP` position/sign reconstruction algorithm:** `Decod_ACELP(sign, index, code)` reconstructs the 40-sample fixed codebook vector from two packed parameters. Extract 4 pulse positions from `index` (13 bits encoding positions in tracks 0-4 with 8 positions each, interleaved step=5): `i0 = (index & 0x7)*5`, `i1 = ((index>>3) & 0x7)*5 + 1`, `i2 = ((index>>6) & 0x7)*5 + 2`, `i3 = ((index>>9) & 0x7)*5 + j` where `j` is 3 or 4: if the 4th pulse's sign bit (bit 3 of the `sign` parameter) is 0, `j = 3` (track 3); if 1, `j = 4` (track 4). Apply signs from `sign` (4 bits, one per pulse): bit set = +1.0, bit clear = -1.0. Place unit pulses at the decoded positions in the zero-initialized `code[L_SUBFR]` vector. Reference: `DE_ACELP.C`
- `bad_lsf` field: external hook initialized to 0, never set by codec itself -- OR'd with BFI for LSP decoding
- Conformance gate: all 10 Annex A decoder test vectors (including undocumented TEST) must pass bit-exactly
- Post-filter pitch search range bounding: `t0_min = T0 - 3`, `t0_max = t0_min + 6`, clamped to PIT_MAX only (no explicit PIT_MIN check; lower bound is implicitly safe because minimum decoded pitch delay is PIT_MIN=20, giving `t0_min >= 17`). Reference: `postfilt.c:124-129`. Documents the exact pitch post-filter search range behavior
- **`pit_pst_filt` 3 dB bypass threshold** (`postfilt.c:296-304`): The pitch post-filter is bypassed when prediction gain < 3 dB. The test is: `L_mult(cmax, cmax) - L_shr(L_mult(en, en0), 1) < 0`, i.e., `cor_max^2 < 0.5 * ener * ener0`. There is no named constant — the 3 dB threshold is implicit in the `L_shr(..., 1)` (factor of 0.5). All three values (`cmax`, `en`, `en0`) are jointly normalized to 16-bit via `norm_l` before comparison (`postfilt.c:288-291`). When `Vad == 0` (DTX inactive frame), pitch PF is bypassed entirely (`postfilt.c:148-151`), independently of this threshold
- State management accessors: `Lsp_decw_reset()`, `Get_decfreq_prev()`, `Update_decfreq_prev()` in `LSPDEC.C` become methods on `DecoderState` in Rust (no standalone functions)
- **`Init_Post_Process()` elimination** (`post_pro.c`): Called from `decoder.c:main()` (not from `Init_Decod_ld8a()`). Zeros the post-processing HP filter state (y1_hi, y1_lo, y2_hi, y2_lo, x0, x1). In Rust, eliminated — initial values in `DecoderState::new()` (see decoder init table row 26)
- **`prev_lsp` vs `lsp_old` distinction** (`LSPDEC.C`): `prev_lsp[M]` (initialized from `freq_prev_reset`) and `lsp_old[M]` serve different purposes. `lsp_old` is the previous frame's decoded LSP vector (used for interpolation). `prev_lsp` is specifically used by the frame erasure path in `Lsp_iqua_cs()` to extract the MA prediction residual from the last good frame.
